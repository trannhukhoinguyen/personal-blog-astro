<H2>Cons of CSS</H2>

- Repeating same properties again & again. In my previous company we've used `CSS` & `SASS` for styling. We've declared some common classes. Still we ended up create new classes which've same properties!
- Coming-up with different class names for styles to not overlap
- Increased bundle size because of repeating the same properties again & again!

<H2>Pros of TailwindCSS</H2>

- It' a combination of utility classes, the CSS properties are tailwind classes exampleğŸ‘‡

```html
<!-- Here i've 2 div's styled with CSS & TailwindCSS -->

<!-- This is Traditional CSS -->
<style>
  .container {
    display: flex;
    align-items: center;
    justify-content: center;
  }
</style>

<div class="container"></div>

<!-- This is Tailwind classes -->
<div class="flex items-center justify-center"></div>
```
